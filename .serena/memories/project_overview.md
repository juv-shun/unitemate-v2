# プロジェクト概要

## プロジェクト名
ユナメイト（unitemate-v2）

## 目的
ポケモンユナイトのドラフトピック形式のマッチを提供するWebアプリケーション。

## 主要機能
- **マッチング**: 10人（5vs5）のプレイヤーをマッチング
- **ドラフト**: BAN/PICK形式でポケモンを選択
  - 3BAN制（各チーム3、計6）
  - PICK順：先攻1 → 後攻2 → 先攻2 → 後攻2 → 先攻2 → 後攻1
  - 各BAN/PICK 15秒の制限時間
- **ロビー共有**: ホストがロビーIDを入力し、全員がゲーム内で参加
- **試合実施**: アプリ外（ゲーム内）で実際の試合を行う

## 開発フェーズ
- **フェーズ1（MVP）**: ランダムマッチ版（レーティングなし）
  - **フェーズ1.3（✅ 実装完了 2026-01-18）**: マッチ成立とチーム分け
    - 10人でマッチ成立（バックエンドで自動マッチング）
    - チーム分け（5人ずつ）と先攻/後攻の決定
    - マッチ成立画面（P1-03）の実装
    - Firebase Functions（1分間隔の定期実行）によるマッチング処理
- **フェーズ2（拡張）**: ランクマッチ版（Eloレーティング導入）
- **フェーズ3（拡張）**: ドラフトピックのモジュール化

## 対象ユーザー
ポケモンユナイトプレイヤー（10人単位でのマッチング）
