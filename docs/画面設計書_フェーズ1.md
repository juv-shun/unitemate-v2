# ユナメイト 画面設計書（フェーズ1）

本書はフェーズ1（MVP: ランダムマッチ）に必要な画面を定義する。

---

## 1. 画面一覧

- P1-01 ログイン/登録
- P1-02 ホーム（インキュー/ステータス）
- P1-03 マッチ成立
- P1-04 ドラフト（BAN/PICK）
- P1-05 ロビーID入力/共有
- P1-06 試合開始案内
- P1-07 設定/プロフィール
- P1-08 エラー/メンテ

---

## 2. 画面詳細

### P1-01 ログイン/登録
- 目的: ユーザー認証
- 主なUI:
  - ログインフォーム（メール/パスワード or プロバイダ）
  - 新規登録導線
- 状態:
  - 入力中 / 認証中 / エラー
- アクション:
  - ログイン
  - 新規登録

### P1-02 ホーム（インキュー/ステータス）
- 目的: キュー参加と待機状況表示
- 主なUI:
  - インキューボタン
  - 待機人数/経過時間
  - キャンセルボタン
- 状態:
  - 未参加 / キュー中 / マッチ成立待ち
- アクション:
  - インキュー開始/停止

### P1-03 マッチ成立
- 目的: マッチ確定の通知とドラフト開始導線
- 主なUI:
  - マッチ成立通知
  - チーム/先攻後攻表示
  - ドラフト開始ボタン
- 状態:
  - 受理待ち / 参加確定
- アクション:
  - 参加確定（参加放棄は原則なし）

### P1-04 ドラフト（BAN/PICK）
- 目的: BAN/PICKを進行
- 主なUI:
  - ターン表示（BAN/PICK, 先攻/後攻, 担当者）
  - ポケモン一覧（検索/フィルタ）
  - クリック→確定ボタン
  - BAN/PICK結果一覧
  - タイマー表示
- 状態:
  - 自分の番 / 他人の番 / 自動確定 / 完了
- アクション:
  - クリックで候補選択
  - 確定
  - リクエスト送信（BAN/PICKリクエスト）
  - リクエスト承認（担当者のみ）

### P1-05 ロビーID入力/共有
- 目的: ホストがロビーIDを入力し共有
- 主なUI:
  - ホスト表示
  - ロビーID入力欄（ホストのみ）
  - ロビーID表示（全員）
  - 入力期限タイマー
- 状態:
  - 待機 / 入力済み / 再抽選 / 無効
- アクション:
  - ロビーID入力（ホスト）

### P1-06 試合開始案内
- 目的: 試合開始手順の確認
- 主なUI:
  - 参加手順説明
  - ロビーID再掲
- 状態:
  - 表示のみ
- アクション:
  - 了解/閉じる

### P1-07 設定/プロフィール
- 目的: 表示名や基本設定の変更
- 主なUI:
  - 表示名編集
  - ログアウト
- 状態:
  - 入力中 / 保存中
- アクション:
  - 保存
  - ログアウト

### P1-08 エラー/メンテ
- 目的: システムエラー時の案内
- 主なUI:
  - エラーメッセージ
  - 再試行ボタン
- 状態:
  - エラー表示
- アクション:
  - 再試行
  - ホームへ戻る

---

## 3. 画面遷移（概要）

- P1-01 → P1-02
- P1-02 → P1-03 → P1-04 → P1-05 → P1-06 → P1-02
- 例外: エラー時は P1-08
